# AGENTS 规范指南

## 技术栈与入口

- 这是一个单包的 Vue 3 + Vite 应用，使用普通 `.js` / `.vue` 文件，不使用 TypeScript。
- 应用启动链路是 `index.html` -> `src/main.js`。
- `src/main.js` 全局安装了 Vue Router 和 Element Plus。
- `src` 目录导入使用 `@` 别名；该别名定义在 `vite.config.js` 中。

## 命令

- 使用 `npm`，不要用 `pnpm` 或 `yarn`；仓库通过 `package-lock.json` 锁定依赖。
- 开发服务器：`npm run dev`
- 构建：`npm run build`
- 预览：`npm run preview`

## 目录规范

### 1. 视图模块（Views）
- 存放路径：`/src/views`
- 按业务模块划分目录，每个模块独立文件夹。
- 模块私有组件放在当前模块的 `components` 目录。

### 2. 布局组件（Layout）
- 存放路径：`/src/layout`
- 存放页面的整体骨架和布局组件。

### 3. 全局通用组件
- 存放路径：`/src/components`
- 用于多个模块复用的全局组件。

### 4. 接口层（API）
- 存放路径：`/src/api`
- 命名规则：`模块名 + Api.js`（例如：`productApi.js`）

### 5. 工具函数（Utils）
- 存放路径：`/src/utils`
- 用于封装可复用的业务逻辑、请求拦截、状态管理、日期格式化等。

### 6. 路由管理
- 存放路径：`/src/router`
- 存放应用的所有路由配置文件。

### 7. 静态资源
- 存放路径：`/src/assets`

## 开发流程

1. 阅读接口文档。
2. 创建 API 文件：在 `/src/api` 下创建对应模块的 Api 文件。
3. 编写视图与组件逻辑：
   - 提取公共逻辑到 `/src/utils` 中。
   - 包括 loading 状态、请求方法（增删改查）、数据变量等。

## 交互规范

- 凡是涉及请求的交互，必须有 `loading` 状态。
- 页面或弹窗中的操作按钮默认需要带 `icon`，不要只放文字按钮。
- 表格操作列中的按钮数量如果大于 `3` 个，不要横向平铺撑宽表格；应收敛为一个下拉选择器，在下拉项中提供对应操作。
- 视图模块中的专属对话框，必须拆分到当前模块的 `components` 目录下，作为该视图的专属组件维护，不要直接把完整对话框写在主视图文件里。
- 对话框组件的内容根节点必须统一设置 `max-height: 80vh`、`overflow-y: auto` 和 `padding-right: 15px`，保证内容过长时可以纵向滚动，且滚动条不要贴近内容区。

## 开发规范

- 凡是前端提交给后端的日期时间字段，统一使用 ISO 风格字符串格式：`YYYY-MM-DDTHH:mm:ss`，不要传 `YYYY-MM-DD HH:mm:ss`。例如：`2026-04-24T00:00:00`。
